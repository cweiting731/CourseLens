# 專題背景與動機

在大學選課時，學生常需要查詢課程是否符合：

- 通識類別是否已經選夠3個種類了
- 必修 / 系外選修 / 系內選修 / 國文 / 通識 / 英文 ...
- 授課老師
- 課程時間、教室
- 學分數
- 等等......

此外，多數系統不支援自然語言搜尋，學生無法直接輸入：
```
「有哪些課是周二下午、屬於通識社會領域的三學分課？」
```
因此，本專題設計一個 **AI 課程屬性查詢 Agent**，讓我們可以使用自然語言即可查詢自己的課程資料，提供更直覺、高效率的選課體驗

# 系統目標
- **自然語言查詢課程資訊**  
    例如：「幫我找通識自然領域、星期三上的課」
- **將使用者的問題轉換成 SQL 查詢**  
    使用 AI 模型（LLM）將自然語言 → SQL
- **資料庫中查詢課程資訊**  
    存放課程代碼、名稱、類別、時間、授課老師等
- **以清楚格式回傳查詢結果**  
    提供給使用者列表式的輸出

# 資料格式
| 欄位名稱        | 資料型態    | 說明                                                                        |
| ----------- | ------- | ------------------------------------------------------------------------- |
| name        | TEXT    | 課程名稱                                                                      |
| category    | TEXT    | 課程分類（必修 / 系內選修 / 系外選修 / 通識 / 英文 / 國文 / 體育）                                |
| credit      | INTEGER | 學分數                                                                       |
| status      | TEXT    | 課程狀態（進行中 / 待定 / 完成）                                                       |
| score       | INTEGER | 修畢分數 (只有 status 為 "完成" 會有修畢分數，無修畢分數以 "-1" 表示)                             |
| teacher     | TEXT    | 授課教師                                                                      |
| generalType | TEXT    | 通識分類（人文學、社會科學、自然與工程科學、生命科學與健康、科際整合領域）只有 category 為 "通識" 時會有通識分類，無內容以空字串表示 |
| detail      | TEXT    | 選填，填寫其他課程內容（附加資訊、上課方式等等），無內容以 空字串 "" 表示                                   |

## Example Record

| ------------ | ---- |
